# Game Design Guardrails

This folder is the source of truth for:

- Game design intent
- Intended social behavior per mode
- Intended social behavior for cross-cutting systems
- Anti-goals and safety boundaries
- Decision-making guardrails for product and implementation changes

## Files

- `GUARDRAILS.md`: global principles and decision rules
- `MODE_PROFILES.md`: mode-by-mode design + social behavior targets
- `SOCIAL_MECHANICS_PROFILES.md`: cross-mode systems (voting, spotlight, queue rules, etc.)
- `MODE_PROFILE_TEMPLATE.md`: template for adding/refining a mode
- `SOCIAL_MECHANIC_TEMPLATE.md`: template for adding/refining a social mechanic
- `INTERVIEW_YES_NO.md`: yes/no interview questions and answer log
- `INTERVIEW_COMPONENTS_YES_NO.md`: yes/no interview for social mechanics/components
- `DECISION_LOG.md`: concrete accepted/rejected design decisions

## How to Use

1. Answer yes/no interview questions in `INTERVIEW_YES_NO.md`.
2. Answer component questions in `INTERVIEW_COMPONENTS_YES_NO.md`.
3. Update `MODE_PROFILES.md` and `SOCIAL_MECHANICS_PROFILES.md` from those answers.
4. Record accepted decisions in `DECISION_LOG.md`.
5. Use `GUARDRAILS.md` to evaluate new feature requests and UI changes.
