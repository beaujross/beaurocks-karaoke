AAHF Festival Co-Host Guide

Keep the line moving. Protect the guest experience.

You are the fast helper layer for the night. You help guests join, find songs, and stay calm while the host keeps approvals, pacing, and stage control.

Tonight's Room
AAHF
When guests need help joining, use the same direct room path every time.

What you own tonight

  • Help guests join fast. Keep the room code and QR story simple.
  • Find the singer first. Every add starts with the right person.
  • Use helper catalog. Build the queue before the room gets rushed.
  • Use co-host votes as planning input. They help the host choose, but they do not auto-change the queue.
  • Watch for confusion. If a guest is stuck, fix the next step, not the whole app.
  • Use Tell Host once. Short, useful, and only when it changes what the host needs to do.

Escalate and vote clean

  • When the host opens a vote, pick one side once. The host sees your choice in the live tally.
  • Votes are bounded. One vote per joined co-host, then the host confirms the winner before anything changes.
  • Tell Host adds context. Use it for issues, not for the vote itself.
  • Audio problem, wrong track, or empty stage. Tell host once.
  • Guest is upset, confused, or thinks they were skipped. Tell host once.
  • No note spam. One clean signal is enough.
  • No second host act. You support the room, not run it.

How your night unfolds

1

Doors open: teach one join story

Say the same thing every time: scan the QR, join AAHF, pick a name, then tap Songs.

2

During rush: build clean queue entries

Find the singer first, then search or browse from the helper catalog so every add lands on the right guest.

3

While the room is live: keep guests oriented

Remind guests to watch their queue, stay nearby when they get close, and keep cheering even when the host pauses for a moment or leaderboard beat.

4

When something matters: signal once

Use Tell Host for audio, pacing, guest issues, or a real queue problem. Do not fill it with chatter or duplicate what the vote already says.
